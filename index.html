            Gatherfield
          

          

            Gatherfield treats node graphs and flow charts as first-class writing surfaces, not diagrams bolted onto notes. The graph is tightly coupled to the plaintext editor: links create structure, panes follow the active note, and versions redraw the map from the text itself.
          

        

      


      

        

          
Connected plaintext

          
The map is part of the editor.

          

            Connected work should not force a choice between text and diagrams. In Gatherfield, plaintext links build the graph, versions change the map, history keeps drafts recoverable, and source material, annotations, outlines, and extracted passages stay close to the notes that use them.
          

        


        

          

            
01 / First-class graph

            
Build the flow chart from the text.

            

              Type plaintext links and Gatherfield creates the nodes, routes the editor panes, and turns branches and returns into a live flow chart.
            

          

          
        


        

          

            
02 / Arrange the map

            
Arrange the graph by hand.

            

              The graph lays itself out automatically. Switch to Manual and drag any node where you want it — order your notes by meaning, not just by their links — and the arrangement saves beside the note.
            

          

          
        


        

          

            
03 / Versioned maps

            
Work on multiple versions of the same note.

            

              Save alternate versions of a note, including the links inside it, and switch between them while each version keeps its own history. The graph updates to match the version you are working on.
            

          

          
        


        

          

            
04 / Plain files and history

            
Recover drafts without hiding the plaintext.

            

              Saved revisions and recovery history stay nearby while the main note remains an inspectable plaintext file.
            

          

          
        


        

          

            
05 / Source references

            
Keep sources close and the map clean.

            

              Citations and source notes stay attached in references while conceptual links define the graph.
            

          

          
        


        

          

            
06 / Comments and highlights

            
Mark up passages without changing the note.

            

              Highlight important text and add comments beside exact lines. The markup lives as annotations, so the note stays readable as plaintext.
            

          

          
        


        

          

            
07 / Draft read-through

            
Edit the outline and the draft follows.

            

              Keep the outline in plaintext beside Draft. Rearrange inclusion links or add another note, save the outline, and the read-through updates to match.
            

          

          
        


        

          

            
08 / Split and combine notes

            
Break a section out, then fold it back in.

            

              Extract a selected section into a new linked plaintext note, then ingest a different linked note to bring that prose back into the active note.
            

          

          
        

      


      

        

          
Built for long-lived projects

          
Keep the workspace useful without locking the notes away.

          

            Because Gatherfield works with plaintext notes, you can use it as your main workspace or bring it in for graphing, versions, references, and drafting. The notes remain ordinary text files, so the project is not locked into one app.
          

        

        

          

            Portability
            
Use Gatherfield exclusively or alongside other editors.

            
Open the same notes in a favorite text editor when that is the right tool, then return to Gatherfield when the project needs a visual map.

          

          

            Shared context
            
Surfaces stay coordinated.

            
The editor, graph, tree, and references point at the same active note instead of becoming separate places to reconcile.

          

          

            Dockable panes
            
Arrange the workspace around the task.

            
Move writing, graph, files, and references into the layout a project needs.

          

          

            Optional metadata
            
Style the map only when it helps.

            
Plaintext remains the body. Frontmatter can add graph presentation details for notes that need them.
